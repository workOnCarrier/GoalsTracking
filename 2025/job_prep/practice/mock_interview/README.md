# Mock Interview Kit

Structure for running the Week 4 mock low-latency C++ interview.

## Tracks
- **Theory Rapid Fire:** Acquire/release, data races, cache lines, CAS semantics.
- **Coding Round:** SPSC queue, Treiber stack with ABA fix, token bucket rate limiter.
- **System Design:** Order ingestion path with SPSC stages, NUMA pinning, RCU config store.
- **Debugging/Optimization:** Diagnose data races and optimize contended counters.

## How To Use
1. Schedule a 60–75 minute block and print `interview_script.md`.
2. Warm up with a 5-minute summary of the week’s goals.
3. Record action items and knowledge gaps under `retro/` (create per session).
